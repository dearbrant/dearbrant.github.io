Things 3

Things 最初遵循 GTD 概念定义产品，而到眼下的 3.0 版本，已经是面向自己的用户群来探讨 “待办事项管理” 这个日常需求，走自己的路，最终形成了自己的产品哲学。

虽然有较复杂的使用逻辑，Things 在呈现上却举重若轻，引导和鼓励你尝试，界面易于上手，信息层级清晰，交互直观，反馈完备，视觉元素细腻精致，完成度极高，整体给人实用、扎实、可靠的使用体验。

Pinterest

使用 Pinterest 的感觉，像在玩游戏。界面全部使用粗号大字体 (远早于 iOS 11)，但凡可以点击的地方 (图片 / 按钮 / 导航 / 文字 / 输入框 / 反馈) 全部被处理得 “对象化”，仿佛游戏界面里的一个个大按钮，吸引人点击，由此 App 整体的操作感非常强，加上 icon 的设计略带卡通童稚的造型 (表意却清晰)，App 便有了独特的视觉语言。

不同界面之间的跳转，因为漂亮的动效实现，仿佛在一层界面里打转一般，消解了 Pinterest 自身因为浏览图片需要频繁跳转所带来的深度导航泥沼，让人浏览发掘到深处，也不觉得有负担，是很漂亮的交互创新。

Spark 2.0

Spark 1.0 邮箱客户端完备的功能，优秀的视觉表达，贴心的个性化设置，令整个 App 成熟优雅，使用起来很舒心，它已经是最值得推荐的邮箱客户端 (我现在用 Edison)。

新近更新的 Spark 2.0，增加了两个厉害的功能——多人实时协作撰写同一封邮件，以及围绕一封邮件进行多人讨论。这两点创新让 Spark 真正具备了可以区别同类工具的、不可替代的产品价值。Email 这种已经超过40年的超稳定产品，Spark 仍旧能做出令人眼前一亮、具备广泛应用价值的创新，令人佩服。

Simplenote

把 “纯文本编辑器” 做到了极致——极致到颇具禅意的，是 Simplenote 。App 的功能界定和界面设计取舍得当，没有一处冗余。视觉层次、颜色、icon 的细节、在不同系统平台上的表现、甚至官方网站和品牌表达，全都干净利落，设计的一致性贯彻得很完整。其设计是着眼于整个 Simplenote 自上而下全局式进行的，这样高的统一性，在数字产品设计领域并不多见。

Gudak

“Preview kills the moment” 这句 slogan 直接为 Gudak 定下了基调，有洞见，因而自有其产品价值；有判断，因而有魅力；有个性，因而有独立作品的精神面貌。对用户心理的洞察而诞生的独特玩法、整体的视觉 / 动效 / 音效 等，已经很好的阐释了自己。

Castor 2.0 / 3.0

作为播客客户端，Castor 借鉴自邮箱的 Inbox 模式，借鉴自 todo list 的整理思维，解决的是重度播客消费者的需求，如果你平日里收听超过10个播客，Castor 的 订阅 > 收集 > 排序 > 收听 的方案是非常有效的，这是它的独创。

Castor 2.0 的播放界面也没有循规蹈矩，它从具体的使用情形出发，重视并突出操作的可用性，将播放界面彻底 “控制面板化”，没有意义不大的封面 (3.0 里提供了封面模式供切换)，全部用于展示核心的操作按钮和相应的区域，仿佛一块 DJ 操控台，这是认真思考，按需出发的设计结果。

Tweetbot for Mac

Tweetbot for Mac 的多栏模式，可以让人同时看到多个主题的 tweet list。这么做有可能是为了方便重度使用者，以便能同时兼顾多频道内容的获取，但这一方式也打破了现在人习惯的、埋首于单向度 news feed 的沉浸式内容浏览方式，多少能削弱一些信息 filter bubble 给人带来的影响，其本身也让 twitter list 的产品价值得以释放。

Bear

Bear 就是我在等待的那个文本编辑器，或者更合适的说法，是满足轻度日常写作的工具。它最核心的价值，就是美。单看它的撰写界面，对快捷方式按钮的处理：充满高级感又文气的 icon、存在感弱却又不至于注意不到的淡色处理、写作状态时隐藏快捷方式、点击后的浮层交互及其界面里的信息设计，无一不是恰到好处，细心之极。

除却美好的使用体验，Bear 在情感化的表达上也下了功夫，比如 user guide 就是用例本身 (Things 3 亦同)，看一遍文档，基本面貌就都摸清了，这种坦诚相待的感觉很好。同时，无处不在的白熊形象，以及软件 Bear 的名称，也拉近了 App 和使用者的距离，非常有效的做法。
